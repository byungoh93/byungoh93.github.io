Breast Cancer Classification¶

The American Cancer Society estimates that about there will be about 276,480 new cases of breast cancer this year and around 42,170 will die from breast cancer in 2020. source

Abnormal growths in the breasts are often classified as either benign or malignant. Most lumps can be classified as benign, and theyre very common and not life threatening. On the other hand, malignant tumors can result in metastatic tumorsspreading beyond the site of origin. These are cancerous tumors that can be life threatening once it metastasizes throughout the body. Therefore, it is important to be able to classify whether a lump is benign or malignant. Patients usually go through several tests before an official diagnosis is made.

There is a type of test using whats called a fine needle aspirate (FNA) that be used to take an image of a cell nucleus from the growth. The properties of the cell nucleus taken from the image can be used to classified whether the cell is benign or malignant. This will be a tutorial will attempt to come up with a model that can classify these growths.

Above is the top 10 rows of the data set. This dataset contains 569 observations and 33 attributes. 'M' for diagnosis means malignant and 'B' means benign. The 10 features and their descriptions provided by the Kaggle page are:
a) radius (mean of distances from center to points on the perimeter)
b) texture (standard deviation of gray-scale values)
c) perimeter
d) area
e) smoothness (local variation in radius lengths)
f) compactness (perimeter^2 / area - 1.0)
g) concavity (severity of concave portions of the contour)
h) concave points (number of concave portions of the contour)
i) symmetry
j) fractal dimension ("coastline approximation" - 1)

The author of the data also provided the "worst" or the largest of these features to the dataset. We will be removing these columns and focus instead of the mean values.

Model selection¶

There are several different models to choose from for this dataset. An obvious choice would be logistic regression since the output is a categorical variable with two outcomes. Before we begin, we need to separate the dependent and independent variables from the training and validation dataframes

In [21]:
x_train = df_train[['radius_mean', 'texture_mean', 'perimeter_mean', 'area_mean', 'smoothness_mean', \
                    'compactness_mean', 'concavity_mean', 'concave points_mean', 'symmetry_mean', 'fractal_dimension_mean']]  
y_train = df_train['diag_bin']

x_valid = df_validate[['radius_mean', 'texture_mean', 'perimeter_mean', 'area_mean', 'smoothness_mean', \
                    'compactness_mean', 'concavity_mean', 'concave points_mean', 'symmetry_mean', 'fractal_dimension_mean']]  
y_valid = df_validate['diag_bin']

The idea behind model selection is taking the training data and coming up with a model that can predict future values. In this case, we use x_train and y_train from the training set to train the model. For each type of model, there are hyperparameters to increase the accuracy of the model. The problem is that it is often difficult to know what the hyperparameter should be set to. Therefore, we will introduce a technique called cross validation that test serveral models with different hyperparameters and choose the one that gives the model the lowest predicted error. Once the model is chosen, it is important to validate the model before sending out into the real world. In this project, we will input x_valid from the validation set into the model and check the output using y_valid.

Conclusion¶

Through model selection using cross validation, we managed to achieve around 90% accuracy in classifying whether a tumor is benign or malignant. At first glance, this doesn't seem very good considering this is an important medical diagnosis. However, I would say it's acceptable considering how the models only has 10 predictors. Also, we only has 569 observation and only half were used to train the model. If we had more data to work with, the models would probably be more accurate and would have less false positive and false negative cases.

From these three models, I would probably choose the logistic regression model because I could easily tune the threshold value for choosing benign or malignant. Right now, it's set to the default value of 0.5, but in the case that I want to reduce the number of false negatives, I could just adjust the value.
